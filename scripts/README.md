# Скрипты мода — QoL и правки перевода

> Игра: Demo **v0.49**.

Набор Ruby-скриптов (RPG Maker VX Ace / RGSS301) для **Russian Translation & QoL Improvement Mod**.
Общий смысл — сделать русскую версию удобнее: чинят интерфейс под кириллицу, добавляют
опции управления и экрана, автосохранение, стелс-хоткей, а также содержат утилиты
для самих переводчиков.

## Как устанавливать

Скрипты вставляются в редактор скриптов игры в раздел **▼ Материалы (Materials)**, выше
**▼ Main**. Файлы с числовым префиксом (`1790`, `1800`, `1810`, `2300`, `2500`) — это
слоты движка, порядок между ними важен (движковые опции грузятся до правок над ними).
Скрипты-утилиты (`=begin ... =end`) по умолчанию **выключены** — их включают вручную
только на время работы над правками.

## QoL и геймплей (активные в игре)

| Файл | Назначение |
|------|------------|
| **WindowFix.rb** | Главный UI-фикс перевода (`IDL-WindowFix`). Расширяет окно описаний до 4 строк, чинит перенос строк по ширине под кириллицу, правит имена персонажей в меню/статусе/сувенирах, «До след. уровня», название карты, обрезанные EX-параметры. Содержит полную русскую таблицу описаний состояний (`StateHelp::Overrides`, ~300 записей) и цветовую отрисовку названий предметов по качеству. |
| **1790 - Ace Menu Engine v1.10.rb** | Yanfly Ace Menu Engine — движок кастомизации главного меню (добавление/перестановка команд, TP/MP-гейджи). База, на которой держатся опции ниже. |
| **1800 - System Options v1.0.rb** | Yanfly System Options (мод EraDKtor) — экран настроек: цвет окон, громкость BGM/BGS/SFX, авто-бег, мгновенный текст, скрытие боевых анимаций. Подключается как пункт меню Ace Menu Engine. |
| **1810 - SystemOptionsPermanent.rb** | Делает громкость и настройки **постоянными между сессиями** — хранит их в `ModLoaderData.rvdata2` вне сейва. Плюс русская локализация всех пунктов System Options и меню титульника. |
| **wasd.rb** | Добавляет в System Options пункт **«Управление»** — переключатель Стрелки ⇄ WASD (само состояние; логику ввода реализует `Lok_Q.rb`). |
| **Lok_Q.rb** | «Unified Input Patch» — собственно перехват `Input`: режим WASD, ремап Q/C, листание страниц, фикс «призрачных» символов движка (чтобы клавиша не дёргала лишнее действие в инвентаре/магазине). |
| **Fullscreen.rb** | Добавляет в System Options пункт **«Экран»** — переключатель Окно ⇄ Полноэкранный (borderless через Win32API). Восстанавливает режим при загрузке сейва. |
| **autosave 2.rb** | Автосохранение по таймеру (каждые 5 мин, 16 слотов, ротация) с защитой: не пишет в бою, диалоге, при запрете сохранений или когда игрок обездвижен. В меню загрузки — переключение обычные файлы ⇄ автосейвы по **Shift**. |
| **Stealth_Hotkey.rb** | Хоткей **E** на карте — быстро активировать «Невидимость»: сначала навык (если хватает MP), иначе расходник, с проигрыванием общего события и уведомлением. |
| **2300 - CustomDialogCmds.rb** | Кастомные управляющие коды в диалогах (`IDL-CustomDialogCmds`). Код `\T[C/L/R]` — выравнивание строки текста по центру/левому/правому краю окна сообщения. |
| **2500 - SpriteDisposeCrashFix.rb** | Загрузчик `SpriteDisposeFix.dll` — фикс случайного вылета движка (RGSS301, `0xC0000005`) при входе в бой из-за освобождения Viewport раньше ссылающихся на него спрайтов. Ставить как можно выше в списке. |

### Что чинит `WindowFix.rb` — по пунктам

- **Термины (`Vocab`):** «До след. уровня» в статусе; префиксы Prico (`Грех` / `Кол-во смертей` / `Стабильность`); `AB_SSS2::NAME1` → «Хранилище».
- **Описание предмета в 4 строки:** `Window_Help` рисует 4 строки вместо 2 — длинные русские описания влезают целиком.
- **Описания состояний:** таблица `StateHelp::Overrides` (~300 записей) с полными RU-описаниями. Имя состояния подставляется из игры (`$data_states`), в таблице — только текст описания.
- **Перенос строк состояний:** `Window_State#wndfix_wrap_text` переносит по ширине окна помощи **реальным** шрифтом (не дефолтным Bitmap-шрифтом), с запасом на хвост — слова не рвутся и не вылезают за край.
- **Цвет названий предметов:** в оригинале имя красилось через `draw_text_ex`, из-за чего движок **не ужимал** текст, если тот чуть не влезал. Здесь покраска по тегу `<品质 N>` (качество) вынесена в частный случай на обычный `draw_text` — текст снова ужимается по ширине.
- **Ширина имени в списках:** `Window_Selectable#item_name_width` + `ITEM_ADD_SPACING` — имя предмета не наезжает на количество; своя ширина имени в `Window_Status`.
- **Опыт до уровня:** `Window_Status#draw_exp_info` — строка «До след. уровня» рядом с «Текущий опыт».
- **Имена в сувенирах/духах:** `Window_SptSpiritList` и `Window_SptSpirit` — правит ширину отрисовки имён (список и описание).
- **Класс/имя актёра в меню:** `draw_actor_class` с шириной 180 — длинное русское имя класса не обрезается.
- **EX-параметры:** `Window_Status` — фикс обрезанных EX-параметров. Переключение страниц в игре уже было; добавлена **подсказка-стрелки** «← N →», показывающая, что страницы можно листать.
- **Название карты:** `Window_MapName#window_width` = половина экрана — русское название карты помещается.
- **Окно выдачи:** `Window_Getinfo#description` вырезает управляющие коды (`\.[...]`) из строки описания.

## Пасхалки

| Файл | Назначение |
|------|------------|
| **Name list.rb** | «Name Reaction Patch» — реакции автора на введённое игроком имя героя. Обычные имена → стёб-диалог; «смертельные» имена → Game Over после реплики; особые имена → смена персонажа. |

## Утилиты переводчика (по умолчанию отключены — тело в `=begin ... =end`)

| Файл | Назначение |
|------|------------|
| **Checking dialogues.rb** | Проверка длины строк во всех картах (диалоги `101/401`, выборы `102`, скролл-текст `105/405`). Строки длиннее `MAX_LEN` (51) выгружаются в отчёт `LongDialogLines.txt`. |
| **comment path.rb** | Одноразовый фикс имён-табличек: проставляет комментарий-имя (`NW...`) над командами показа текста в `Map*/CommonEvents/Troops.rvdata2`, делая `.bak`. Запускается прямо при старте игры (F5) и закрывает её по завершении. |
| **Fix rpg.rb** | Дубль загрузчика `SpriteDisposeFix.dll` (`SpriteDisposeFixLoader`) — вспомогательный/резервный вариант скрипта `2500`. |
